# 국제화 (i18n) 규칙

## 개요

`infrastructure/i18n.py` 가 런타임 번역을 담당합니다.
UI 문자열은 하드코딩하지 않고 반드시 `t()` 함수를 사용합니다.

## 기본 사용법

```python
from calendar_app.infrastructure.i18n import t

# 기본 패턴: t(key, fallback)
label = t("menu.system", "시스템")
title = t("dialog.task_create", "태스크 만들기")
```

`fallback`은 번역 키가 없을 때 표시되는 기본값(한국어)입니다.

## 로케일 파일 구조

```
locales/               ← 번들 로케일 (배포 포함)
  ko.json              ← 한국어 (기본)
  en.json              ← 영어
  ja.json              ← 일본어
  ...
LOCALAPPDATA/DarkCalendar/locales_user/   ← 사용자 오버라이드
  ko.json              ← 사용자 커스텀 한국어 (번들보다 우선)
```

## 로케일 관련 API

```python
from calendar_app.infrastructure.i18n import (
    t,
    list_available_locale_codes,   # 사용 가능한 언어 코드 목록
    get_locale_display_name,       # 코드 → 표시 이름 ("ko" → "한국어")
    resolve_locale_file_path,      # 언어 코드 → 파일 경로
    get_user_locales_dir,          # 사용자 로케일 디렉터리 경로
)
```

## 런타임 언어 전환

`presentation/main_window/top_menus/system_menu.py`의 `build_system_menu()`에서
언어 선택 메뉴를 빌드합니다. 선택 시 `QSettings`에 `lang_code`를 저장하고 재시작 안내.

## 새 번역 키 추가 절차

1. `locales/ko.json`에 한국어 키-값 추가
2. `locales/en.json`에 영어 번역 추가
3. 코드에서 `t("새.키", "한국어 fallback")` 사용
4. 번역이 없는 언어는 `fallback` 값이 표시됨

## 로케일 JSON 형식

```json
{
  "menu.system": "시스템",
  "menu.widget_clock": "디지털 시계",
  "dialog.task_create": "태스크 만들기",
  "label.today": "오늘"
}
```

중첩 객체 없이 플랫한 키 구조를 사용합니다.

## 주의사항

- `t()` 함수는 현재 로드된 로케일에서 키를 검색합니다
- 키가 없으면 `fallback` 반환 (빈 문자열 아님)
- `locales/` 디렉터리의 파일은 `read_text_with_legacy_fallback()`으로 읽습니다 (구버전 인코딩 호환)
- 사용자 로케일 파일이 번들 파일보다 우선합니다 (`prefer_user=True`)
